# Control de versiones

## Esquema

El manual emplea versiones `MAYOR.MENOR.PARCHE`:

- **MAYOR:** cambio metodológico o reorganización incompatible.
- **MENOR:** nueva aplicación, flujo o capítulo relevante.
- **PARCHE:** corrección o aclaración sin alterar la metodología.

## Trazabilidad

Cada publicación estable debe incluir etiqueta Git, GitHub Release, `CHANGELOG.md`, fecha de cierre y relación de versiones de software verificadas.
